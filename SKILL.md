# Novel Style Writer — 网络小说风格仿写

## 执行指令（重要）

**读完本文件后立即开始执行。不要进入 plan mode。不要反复规划。直接按流程产出内容并写入文件。**

如果需要参考文件中的详细说明，在需要时再读取，不要一次性全部读取。

## 项目文件结构（每一步必须保存）

每次创作小说时，必须在工作目录下创建以下项目结构。**每个 Phase 完成后，必须将产出写入对应的文件。**

```
[小说名]/
├── style-profile.md          ← Phase 1: 风格画像
├── concept.md                ← Phase 2: 核心概念
├── world-building.md         ← Phase 3: 世界观构建
├── characters/
│   ├── protagonist-[主角名].md  ← Phase 4: 主角完整卡片
│   ├── antagonist-[反派名].md   ← Phase 4: 反派卡片
│   └── supporting-cast.md       ← Phase 4: 配角+关系网络
├── outline/
│   ├── 总纲.md                  ← Phase 5.1: 总纲
│   ├── 卷纲/                    ← Phase 5.2: 每卷一个文件
│   │   ├── 第一卷-[卷名].md
│   │   ├── 第二卷-[卷名].md
│   │   └── ...
│   ├── 章纲/                    ← Phase 5.3: 每10章一个文件
│   │   ├── 第01-10章.md
│   │   ├── 第11-20章.md
│   │   └── ...
│   ├── 节点纲.md                ← Phase 5.4: 节点纲
│   └── 日常纲.md                ← Phase 5.5: 日常纲
├── trackers/
│   ├── 人物状态.md               ← Phase 6: 人物状态追踪（每章更新）
│   ├── 伏笔追踪.md               ← Phase 6: 伏笔追踪（每章更新）
│   └── 扩展元素.md               ← Phase 3: 扩展元素追踪
├── chapters/
│   ├── 第001章-档案.txt          ← Phase 6: 各章节正文（TXT格式，一章一个文件）
│   ├── 第002章-溺水村（一）.txt
│   └── ...
└── review/
    └── 整体审校.md               ← Phase 6.5: 整体审校报告
```

**每个 Phase 的写入规则**：
- Phase 1 完成后 → 写入 `style-profile.md`
- Phase 2 完成后 → 写入 `concept.md`
- Phase 3 完成后 → 写入 `world-building.md` + `trackers/expansion.md`
- Phase 4 完成后 → 写入 `characters/` 目录下的各个文件
- Phase 5 完成后 → 写入 `outline/` 目录下的各个文件
- Phase 6 每章完成后 → 写入 `chapters/` + 更新 `trackers/` 下的追踪文件
- Phase 6.5 完成后 → 写入 `review/final-review.md`

## 核心原则（按优先级排序）

### ⚠️ 最高优先级：纲要一致性准则

**当用户提出修改建议时，必须同步更新所有相关纲要文件**，不能只改一个文件而遗漏其他文件。

**修改时必须检查的文件清单**：
- `outline/总纲.md` — 全书宏观骨架
- `outline/卷纲/` — 每卷详细规划
- `outline/章纲/` — 每章执行计划
- `outline/节点纲.md` — 关键转折点
- `outline/日常纲.md` — 过渡章模板

**修改传播规则**：
- 修改世界观 → 更新总纲、所有卷纲、所有相关章纲
- 修改人物设定 → 更新所有涉及该人物的章纲、节点纲
- 修改情节 → 更新总纲、相关卷纲、相关章纲、相关节点纲
- 修改副本设定 → 更新该副本的所有章纲、相关日常纲

### 创作时的参考优先级（严格遵守）

**写作每一章时，必须严格按以下优先级参考纲要**：

1. **章纲**（最高优先级）— 每章的具体内容、事件、钩子、情绪弧、爽点类型
2. **节点纲** — 关键转折点的超详细设计（对话、动作、情绪节拍）
3. **日常纲** — 过渡章的模板化写法
4. **卷纲** — 本卷的整体规划和副本安排
5. **总纲** — 全书宏观骨架和悬念分层

**⚠️ 严格规则**：
- 每章写作前**必须先读取对应的章纲**，按章纲的核心事件、情绪弧、钩子、爽点来写
- 章纲中规定的核心事件**必须全部包含**，不能遗漏
- 章纲中规定的钩子类型和强度**必须匹配**
- 章纲中规定的爽点类型和位置**必须匹配**
- 不能自行增删章纲中未规划的内容
- 如果发现章纲有问题，**先修改章纲，再写作**，不能跳过章纲直接写

---

### 其他原则

1. **前置构建优先**：大纲→卷纲→章纲全部完成后才动笔。绝不跳步。
2. **每层确认**：每个阶段产出必须经用户确认后才进入下一阶段。
3. **风格一致性**：全程以风格画像为锚点。
4. **中文为主**：所有输出默认中文。
5. **网络小说特性**：以商业网文标准创作，注重可读性、爽感、节奏和读者留存。

## 准备时长指导（大神经验）

| 阶段 | 时长 | 内容 |
|------|------|------|
| Phase 1 风格分析 | 1-3天 | 分析参考文本，生成风格画像 |
| Phase 2 核心概念 | 3-7天 | 回答五个问题，确定方向 |
| Phase 3 世界观构建 | 1-2周 | 力量体系、地理、势力、历史 |
| Phase 4 人物体系 | 3-7天 | 主角传记、反派、配角、关系网 |
| Phase 5 大纲细化 | 1-2周 | 总纲→卷纲→章纲→节点纲 |
| Phase 6 黄金三章 | 3-7天 | 写作+打磨前三章 |
| **总计** | **4-8周** | 取决于题材复杂度 |

**注意**：悬疑类（辰东式）需要更长的伏笔规划时间；世界观复杂的作品（远瞳式）需要更长的世界构建时间。

## 网络小说核心要素

网络小说与传统文学小说有本质区别。创作时必须牢记以下要素：

### 黄金三章

前三章决定读者是否继续阅读。必须做到：
- 第一章：建立主角 + 核心冲突 + 世界设定，**300字内必须出现第一个钩子**（悬念/冲突/异常），不能全是日常铺垫
- 第二章：冲突升级 + 第一个爽点/悬念
- 第三章：主角展现能力/特质 + 更大的悬念铺开

**开头节奏公式**：
- 第1段：一个抓人的画面/悬念/冲突（不超过3句）
- 第2-10段：快速建立人物和环境（用对话推进，不用大段描写）
- 第11段起：核心事件触发

### 爽点分布

每章至少包含 1 个爽点或情绪高点。**爽点必须有"蓄力→释放"的过程**，不能只有压抑没有释放：
- **打脸**：被人小看后实力碾压（蓄力：被嘲讽/轻视 → 释放：碾压回去）
- **升级**：突破、获得新能力、解锁新装备（蓄力：困境/瓶颈 → 释放：突破）
- **揭秘**：真相揭露、伏笔回收、身份曝光（蓄力：疑问/悬念 → 释放：答案揭晓）
- **反转**：出乎意料的剧情转折（蓄力：读者以为A → 释放：其实是B）
- **情感**：师徒、兄弟、恋人之间的情感爆发（蓄力：压抑/误解 → 释放：和解/表白）
- **装逼**：低调展示实力，震惊旁人（蓄力：隐藏实力 → 释放：不经意展露）

**特别注意**：如果全章基调偏压抑/悬疑，爽点可以是"揭秘型"——读者和主角一起拼出真相的那一刻就是爽点。但必须让读者感觉到"啊，原来如此"的智识快感，不能只是更多压抑。

### 章末钩子（必须）

每章结尾必须有钩子，类型包括：
- 悬念型："门外传来脚步声。"（不揭示来者）
- 反转型：本章建立的认知被颠覆
- 危机型：主角陷入困境或新威胁出现
- 揭秘型：即将揭露重要信息但卡在关键处
- 情感型：关系出现重大转折

### 节奏控制

- **网文节奏偏快**：少铺垫多推进，每 2000-3000 字必须有一个事件
- **对话占比偏高**：约 30-40%，对话是推进情节和展现人物的主要手段
- **段落偏短**：适配手机阅读，单段不超过 4-5 行
- **描写克制**：环境描写不超过 2-3 句，服务于氛围而非炫技

### 人物塑造

- **主角必须有记忆点**：一个鲜明的性格标签 + 一个独特的能力/习惯 + 一句标志性台词或口头禅
- **配角也必须有记忆点**：每个配角至少一个让人记住的特征（说话方式/小动作/独特外貌），不能只是功能性工具人
- **反派要有魅力**：不能只是"坏"，要有自己的逻辑和吸引力
- **人物通过事件展现**：少用内心独白，多用行动和对话
- **人物出场要有"亮相"**：每个重要角色第一次出场时，用一个有画面感的细节让人记住ta

## 副本创作标准（无限流/恐怖类适用）

写无限流或恐怖类小说时，每个副本必须遵守以下标准：

### 1. 规则发现方式
- **通过亲眼看到后果发现**，不是被告知
- 例如：亲眼看到有人违规后被拖入水中死亡，而不是有人告诉你"不要在水里待超过3分钟"

### 2. 时限机制
- 每个副本有**明确的死亡时限**（如7天、24小时）
- 时限到期未完成任务 = 死亡
- 时限制造持续的紧迫感

### 3. 参与人数
- 每个副本 **5-6人团队**，不是只有主角
- 不同角色有不同的能力、性格、立场
- 团队互动是恐怖感和人物塑造的重要来源

### 4. 配角定位
- **不全知**：经验丰富但有自己的恐惧、弱点、秘密
- **不主动教**：通过行动展示，而非口头讲解
- **有问题**：每个配角都有自己的困境（如绝症、倒计时、心结）

### 5. 信任建立
- **逐步建立**：通过共同经历事件才建立信任
- **有冲突**：团队内部有分歧、猜忌、利益冲突
- **有牺牲**：有人为团队牺牲时，信任才真正建立

### 6. 探索地图
- **多区域**：不是搜几间房，而是多个不同区域
- 例如：祠堂、水井、后山、渡口、废弃民居
- 每个区域有不同的恐怖元素和线索

### 7. 恐怖体现
- **通过事件体现**：主角亲身经历恐怖事件
- **不是氛围描写**：而是"亲眼看到有人被拖走""差点被抓住"
- **金手指有代价**：每次使用能力都有副作用

### 8. 副本结构模板
```
第1-2章：进入副本 + 建立规则（亲眼看到后果）
第3-5章：多区域探索 + 收集线索
第6-8章：遭遇危险 + 团队互动
第9-11章：使用金手指 + 推理死因
第12-14章：找到尸体 + 揭示真相
第15-16章：副本崩溃 + 回归 + 代价揭示
```

## 六阶段流程

### Phase 1: 风格分析

用户提供参考文本 → 分析 9 个维度 → 生成风格画像 → 用户确认

**9 个维度**：叙事视角、句式特征、词汇特征、对话风格、节奏与留白、修辞手法、氛围营造、人物塑造、主题与结构

**每个维度输出**：定性描述 + 定量数据 + 原文示例（1-3段）

**风格适配原则**：参考文本优先，网文基线兜底。参考文本有明确偏好的，跟随参考文本；参考文本未覆盖的维度（如爽点分布），使用网文基线。详见 `references/style-analysis-guide.md` 开头的「风格适配机制」。

**输出**：融合风格画像文档（参考文本风格 + 网文商业节奏），用户确认后作为后续所有阶段的锚点。

**文件写入**：用户确认后，将风格画像写入项目目录的 `style-profile.md`。

---

### Phase 2: 核心概念定型

与用户确认以下内容（参考大神们的"五个必须回答的问题"）：

**五个必须回答**：
1. **核心爽点是什么？** — 打脸？升级？揭秘？反转？整本书围绕这个爽点构建
2. **力量/成长体系怎么设计？** — 等级、突破条件、能力差异、代价（这是故事引擎）
3. **金手指是什么？** — 主角的独特优势，必须：独特、有限制、能扩展、能生成故事
4. **核心冲突是什么？** — 能撑百万字的矛盾，必须可分层、可升级
5. **情感弧线怎么走？** — 读者的情绪旅程

**额外确认**：
- **题材类型**：大类型（玄幻/仙侠/都市/科幻/历史/悬疑/言情/武侠/末世/异世界）+ 子类型 + 调性
- **核心主题**：小说探讨的核心问题（猫腻式：先定主题，一切服务主题）
- **一句话前提（Logline）**：当 [主角] 遭遇 [事件]，ta 必须 [行动]，否则 [代价]
- **爽点设计**：本书的核心爽点类型（升级流/扮猪吃虎/种田/无限/系统/……）
- **目标读者**：男频/女频、年龄段、阅读偏好

**商业化验证**（编辑视角）：
- 概念能否用一句话让读者想看更多？
- 开头300字内是否有钩子？
- 金手指是否与市面作品有差异化？
- 这个概念能撑100-300万字不崩吗？

用户确认后进入下一阶段。

**文件写入**：用户确认后，将核心概念写入项目目录的 `concept.md`。

### Phase 3: 世界观构建

如需详细清单，读取 `references/world-building-guide.md`。

**构建优先级**（大神共识：力量体系第一）：

| 优先级 | 内容 | 原因 |
|--------|------|------|
| 1 | **力量/修炼体系** | 这是故事引擎，一切围绕它运转 |
| 2 | **地理与时代背景** | 决定故事的"味道"和冲突范围 |
| 3 | **势力分布与关系** | 主角周旋的政治格局 |
| 4 | **经济/社会体系** | 世界的质感和逻辑自洽 |
| 5 | **历史与谜团** | 长线叙事钩子（辰东式） |
| 6 | **文化风俗** | 深度和差异化 |

**力量体系设计七大原则**（大神共识）：
1. **等级清晰**：每个等级有明确名称和量化标准
2. **突破仪式感**：每次突破都是重要爽点
3. **能力差异化**：不同等级有质的区别，不是数值提升
4. **可视化**：力量表现要能被读者"看到"（颜色、形态、范围）
5. **名称朗朗上口**：读者能记住、能传播
6. **代价机制**：力量必须有代价，才能创造张力
7. **故事引擎**：体系本身要能生成冲突和机遇

**"宇宙分层法"**（我吃西红柿式，适用于需要宏大世界观的作品）：
- 世界由多个层次组成（如：凡人界→仙界→神界）
- 每层有新的力量体系、势力、规则
- 主角每突破一层，读者视野随之扩大
- 天然产生持续新鲜感和升级爽感

**必填项**：
- 力量/规则体系（等级划分、突破条件、上限设定、升级爽感）← 最优先
- 时空背景（时间线、空间设定、历史年代感）
- 社会体系（政治体制、社会阶层、经济、文化）
- 势力阵营（主要势力、势力间关系、势力与主角的关系走向）
- 历史脉络（重大事件、传说秘辛、遗留问题）

**网络小说特别注意**：
- 力量体系必须清晰可量化，让读者能感受到"变强"
- 等级划分要有仪式感（突破场景、天地异象、实力碾压）
- 世界观要为"升级"和"冲突"留出足够空间

**扩展预留机制**（防止写到后面没素材）：

在初始构建世界观时，必须预留以下扩展空间：

| 扩展维度 | 预留方式 | 示例 |
|---------|---------|------|
| **地理扩展** | 核心区域之外标注"未探索区域"，名称+一句话描述 | 主舞台江南，但提到"关外""海外""南疆"等未展开区域 |
| **势力扩展** | 除了详细势力，列出2-3个"传闻中的势力"，只给名字和模糊印象 | 详细写5个势力，另外提到"西域有个神秘门派""海上有海盗王" |
| **力量扩展** | 最高等级之上留一个"传说级"，不定义具体能力 | 写了5级体系，第5级"天人"只说"传说中存在，当世无人达到" |
| **人物扩展** | 核心角色之外，预留"未出场重要人物"的名字和身份线索 | 详细写10个角色，另外提到"天机阁还有三位长老从未露面" |
| **历史扩展** | 历史时间线上标注"未解之谜"，留待后续揭示 | 青衣巷事件只给结果不给全貌，真相分多卷逐步揭露 |
| **道具扩展** | 关键物品只描述部分功能，后续揭示隐藏用途 | 名册只说"记录着暗桩身份"，后续揭示"还有第三页" |

**扩展节奏**：每卷新揭示1-2个预留元素，不能一次性全部展开。详见 `references/world-expansion-guide.md`。

用户确认后定稿。

**文件写入**：用户确认后，将世界观构建写入 `world-building.md`，同时创建 `trackers/expansion.md` 记录扩展预留元素。**在写入前，确保 `trackers/` 目录已创建**。

### Phase 4: 人物体系构建

如需人物卡片模板，读取 `references/character-template.md`。

**人物设计顺序**（大神共识）：力量体系 → 人物 → 大纲（猫腻例外：主题→人物→世界观）

**主角设计——"人物传记"法**（猫腻式）：

为每个重要角色写一份500字以上的"人生传记"，包含：
- 出生背景、成长环境
- 性格形成的关键事件
- 人生转折点
- 内心深处的恐惧和渴望
- 行为逻辑的根源

传记不是给读者看的，是给作者自己用的——确保人物行为始终符合其"人生经历"。

**主角设计——"标签+反差"法**（烽火戏诸侯式）：
- 每个人物有一个鲜明标签（如：纨绔、冷酷、话痨）
- 同时有一个反差（纨绔外表下的深沉、冷酷外表下的温情）
- 反差让人物立体，标签让人物好记

**主角设计——"30秒法则"**（唐家三少式）：
> "每个人物出场时，读者必须能在30秒内记住他。要么靠外貌，要么靠性格，要么靠能力。最好三个都有。"

**"可控不安全感"原则**（猫腻核心方法）：
- 主角必须**足够安全**让读者放心追读（有保护者、有底牌、有能力）
- 同时**足够危险**让读者有紧张感（保护者会离开、底牌有限、能力有上限）
- 庆余年范闲的例子：父亲是皇帝、母亲是叶轻眉、养父是户部尚书、陈萍萍保护 → 但这些保护者之间互相猜忌
- 结果：读者感到紧张但信任作者

**最小人物集**（动笔前必须完成）：

| 角色类型 | 深度要求 | 数量 |
|---------|---------|------|
| 主角 | 完整传记（500字+） | 1 |
| 主要反派 | 完整传记 | 1-2 |
| 核心配角 | 角色卡片 | 3-5 |
| 第一弧配角 | 简要笔记 | 3-5 |
| 导师/权威人物 | 角色卡片 | 1-2 |

**配角设计**（5-15人）：简化卡片——姓名、身份、核心性格、故事功能、出场/退场

**反派设计**：同主角级别深度——反派逻辑、与主角的镜像关系、魅力点、底线

**关系网络**：所有角色关系 + 关系变化节点

**网络小说特别注意**：
- 主角的"金手指"（独特优势/能力/系统）必须明确
- 人物弧光要配合"爽点"——每次成长都伴随一个爽感场景
- 女频注重情感关系发展，男频注重实力成长和地位提升

用户确认后定稿。

**文件写入**：用户确认后，将人物体系写入 `characters/` 目录。**在写入前，确保 `characters/` 目录已创建**。
- 主角完整卡片 → `characters/protagonist-[主角名].md`
- 反派卡片 → `characters/antagonist-[反派名].md`
- 配角+关系网络 → `characters/supporting-cast.md`

### Phase 4.5: 容量检查（每卷开写前必做）

**在写每一卷之前**，必须做容量检查，确保内容足够撑满目标章数×目标字数。

**容量检查公式**：
```
本卷内容量 = 副本数 × 每副本预估章数 + 日常章数 + 过渡章数

如果 < 目标章数 → 需要增加副本或扩展剧情
如果 > 目标章数 → 需要精简或拆分到下一卷
```

**容量检查清单**：

| 检查项 | 标准 | 说明 |
|--------|------|------|
| 副本数量 | 5-6个 | 每个副本至少5章 |
| 日常章数量 | 不超过总章数的30% | 避免拖沓 |
| 每个副本的情节量 | 至少3个关键事件+1个反转 | 撑满5-8章 |
| 人物发展 | 每卷至少2个角色有弧光推进 | 避免扁平 |
| 悬念推进 | 每卷至少揭晓1个A级悬念 | 保持吸引力 |
| 爽点分布 | 每5-8章至少1个S/A级爽点 | 保持节奏 |

**容量不足时的扩充方法**：
- 增加副本（最有效）
- 扩展副本内部情节（增加反转、增加团队互动）
- 增加有信息量的日常章（推进伏笔、深化人物）
- 增加支线剧情（配角个人故事）

**容量过多时的精简方法**：
- 合并相似副本
- 压缩日常章（去除重复、精简描写）
- 将次要情节移到下一卷

**文件写入**：容量检查结果记录在卷纲文件的开头。

### Phase 5: 大纲细化（核心阶段）

如需详细写作指南和JSON模板格式，读取 `references/outline-structure-guide.md`。

大纲采用**5层细化体系**（大神共识）：

| 层级 | 名称 | 范围 | 典型长度 |
|------|------|------|---------|
| 第1层 | 总纲（大纲） | 全书宏观骨架 | 3000-10000字 |
| 第2层 | 卷纲（细纲） | 每卷详细规划 | 每卷1000-3000字 |
| 第3层 | 章纲 | 每章执行计划 | 每章200-500字 |
| 第4层 | 节点纲 | 关键转折点的超详细设计 | 视情况 |
| 第5层 | 日常纲 | 过渡章的模板化写法 | 标准模板 |

**文件写入**：每个层级确认后写入 `outline/` 目录。**在开始 Phase 5 之前，必须先创建 `outline/` 目录**（如果不存在）。

- 5.1 总纲 → `outline/总纲.md`
- 5.2 卷纲 → `outline/卷纲/` 目录，**每卷一个文件**：
  - 第一卷 → `outline/卷纲/第一卷-[卷名].md`
  - 第二卷 → `outline/卷纲/第二卷-[卷名].md`
  - 以此类推
- 5.3 章纲 → `outline/章纲/` 目录，**每10章一个文件**：
  - 第1-10章 → `outline/章纲/第01-10章.md`
  - 第11-20章 → `outline/章纲/第11-20章.md`
  - 第21-30章 → `outline/章纲/第21-30章.md`
  - 以此类推
- 5.4 节点纲 → `outline/节点纲.md`
- 5.5 日常纲 → `outline/日常纲.md`

**重要**：每个层级完成后必须**立即**写入文件，不要等到全部完成再保存。用户确认后才进入下一层级，但文件写入在用户确认时完成。

#### 5.1 总纲 → 用户确认

- 结构选择（三幕式/起承转合/英雄之旅/多线并行）
- 主线冲突链（冲突→升级→高潮→解决）
- 副线设计（2-3条，含交汇点和解决时机）
- 全书情绪曲线（波浪形，波峰递进，波谷不归零）
- 关键转折点（5-8个）
- 结局设计
- **爽点节奏图**：标注全书的爽点分布，确保不会连续多章无爽点
- **悬念分层**（辰东式）：
  - S级悬念（全书级）：贯穿全书的核心谜团
  - A级悬念（卷级）：每卷的核心谜团
  - B级悬念（章级）：每几章的小悬念
  - C级悬念（场景级）：单章内的微悬念

**节奏公式**（天蚕土豆式"三章一流程"）：
| 章节 | 内容 | 情绪 |
|------|------|------|
| 第1章 | 进入新环境/遇到新挑战 | 压抑、期待 |
| 第2章 | 准备/蓄力，展示部分实力 | 紧张、期待 |
| 第3章 | 全力爆发，打脸/突破 | 爽、释放 |

#### 5.2 卷纲 → 用户逐卷确认

每卷包含：
- 卷名与主题、卷级目标、关键事件（3-5个）
- 新出场/退场人物、推进的副线
- 情绪曲线、卷末钩子、预估字数
- **本卷爽点分布**：标注每卷的高潮和爽感节点
- **本卷悬念清单**：S/A/B/C各级悬念的状态

#### 5.3 章纲 → 用户确认（可选逐章或批量）

每章包含以下全部字段：

**基础层**：章节编号、标题、所属卷、视角人物、场景地点、故事内时间

**人物层**：出场人物列表、重点刻画人物、关系变化

**情节层**：关键事件、核心冲突（内部/外部/双重）、冲突解决方式、副线推进、章末钩子（必须有）

**结构层**：开篇方式、收束方式、场景分解、节奏、叙事手法

**情绪氛围层**：情绪弧、基调、感官侧重（视/听/触/嗅/味）、氛围关键词

**伏笔呼应层**：埋下的伏笔（含计划回收章节）、回收的伏笔、象征/意象

**风格层**：风格侧重、对话占比（网文建议30-40%）、描写密度、目标字数（网文建议3000-5000字）

**爽点层**：本章爽点类型、爽点位置（开头/中段/结尾）、爽感强度（1-5）

**钩子强度**：S级（必须翻页）/ A级（强烈想翻）/ B级（好奇）/ C级（轻微）

#### 5.4 节点纲（大转折点特别规划）

对以下关键节点，需要写**超详细的节点纲**（比普通章纲详细3-5倍）：
- 主要转折点
- 高潮场景
- 核心悬念揭晓
- 主角突破/觉醒
- 反派对决

节点纲包含：
- 具体对话要点
- 动作/战斗细节
- 情绪节拍（每500字的情绪变化）
- 旁观者反应设计
- 金句/名场面设计

#### 5.5 日常纲（过渡章模板）

日常、修炼、铺垫等过渡章，使用标准化模板：
- 日常章的固定结构（互动→铺垫→钩子）
- 信息释放节奏（每章至少一个新信息）
- 人物塑造机会（利用日常展现人物性格）
- 避免"水章"的检查项

### Phase 6: 逐章创作与审校

**文件写入**：
- 每章正文 → `chapters/` 目录，**TXT格式，一章一个文件**：
  - 命名规则：`第[XXX]章-[章节标题].txt`
  - 示例：`第001章-档案.txt`、`第002章-溺水村（一）.txt`
  - **重要**：每写完一章，立即保存为TXT文件，不要等全部写完再保存
- 人物状态追踪 → `trackers/人物状态.md`（每章更新）
- 伏笔追踪 → `trackers/伏笔追踪.md`（每章更新）
- 整体审校 → `review/整体审校.md`（全书完成后）

**项目初始化**：进入 Phase 6 前，先创建以下追踪文件：
- `trackers/character-state.md`（初始状态）
- `trackers/foreshadowing.md`（初始伏笔列表）

#### 6.0 写前准备（每章必做）

**在写每一章之前**，必须回查以下文档，确保前后一致：

1. **回查世界观文档**：本章是否涉及新地点/势力/规则？如果有，检查是否与已有设定矛盾
2. **回查人物状态表**：各角色当前的状态（位置、知道什么、持有物品、伤势、情绪）是否正确
3. **回查章纲**：本章的伏笔任务（要埋什么、要回收什么）是否清楚
4. **回查前一章**：上一章结尾的状态，本章开头要无缝衔接

如需人物状态追踪模板，读取 `references/character-state-tracker.md`。

#### 6.1 写作
- 严格遵循章纲
- 以风格画像为锚点
- 参考前几章保持衔接
- 注意伏笔埋设和回收
- **每章必须有章末钩子**
- **每章至少一个爽点或情绪高点**
- **段落适配手机阅读，不超过4-5行**
- **引入新设定时必须检查世界观文档**

#### 6.2 写后更新（每章必做）

每章完成后，**必须更新以下文档**：

1. **更新人物状态表**：记录本章结束后各角色的状态变化
2. **更新世界观扩展记录**：如果本章引入了新地点/势力/规则，追加到世界观文档
3. **记录伏笔状态**：本章埋了哪些伏笔、回收了哪些伏笔，更新伏笔追踪表

如需伏笔追踪模板，读取 `references/foreshadowing-tracker.md`。

#### 6.3 风格自检
每章完成后回溯风格画像，检查句式、对话、用词、视角、节奏是否一致。偏离则修正。

**网文特别自检**：
- [ ] 章末是否有钩子？
- [ ] 是否有爽点/情绪高点？
- [ ] 段落是否适合手机阅读？
- [ ] 对话占比是否在30-40%？
- [ ] 是否有连续超过500字无事件推进？

**一致性自检**：
- [ ] 本章是否引入了新设定？是否与世界观文档一致？
- [ ] 人物言行是否与性格设定一致？
- [ ] 力量表现是否与等级设定一致？
- [ ] 时间线是否连续？
- [ ] 人物称呼/外貌描写是否前后一致？

#### 6.4 用户审阅
用户通过 → 下一章。用户提意见 → 修订。用户要求改章纲 → 回到5.3。

#### 6.5 整体审校
全书完成后检查：连贯性、伏笔回收、人物一致性、风格一致性、情绪曲线、副线收束。

如需审校清单，读取 `references/quality-checklist.md`。
